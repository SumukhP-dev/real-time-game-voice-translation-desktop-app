# Video Recording Guide for 3-Minute Demo

## Equipment Needed
- Screen recording software (OBS Studio, Camtasia, or similar)
- Microphone for narration
- CS:GO 2 game (or similar competitive game)
- The CSGO2 Voice Translation application

## Pre-Recording Setup

### 1. IBM watsonx Orchestrate Environment
- Set up your watsonx workspace with the translation workflow
- Have the workflow dashboard ready for demonstration
- Ensure all integrations are configured and working

### 2. Application Setup
- Install and configure the CSGO2 Voice Translation Mod
- Test all audio capture methods (VB-Audio Cable recommended)
- Verify translation services are running
- Prepare sample audio files in multiple languages

### 3. Game Environment
- Have CS:GO 2 (or similar) running in windowed mode
- Prepare gameplay scenarios that demonstrate communication barriers
- Have teammates ready with different language samples (or use pre-recorded audio)

## Recording Structure

### Scene 1: Introduction (0:00 - 0:30)
**Visual Setup:**
- Split screen: Left side showing gaming communication problems, right side showing solution overview
- Include statistics about multilingual gaming communities
- Show the application logo and IBM watsonx branding

**Recording Tips:**
- Use professional background music (subtle)
- Ensure clear, high-quality narration
- Keep text overlays readable and concise

### Scene 2: Problem Demonstration (0:30 - 1:00)
**Visual Setup:**
- Full screen gameplay recording
- Show teammates speaking different languages without translation
- Highlight missed callouts and poor coordination
- Use on-screen text to show what's being said vs. what's understood

**Recording Tips:**
- Capture actual gameplay footage
- Use subtitles to show language barriers
- Emphasize the frustration and confusion

### Scene 3: Solution Demonstration (1:00 - 2:30)
**Visual Setup:**
- Application interface demonstration
- IBM watsonx Orchestrate dashboard
- Real-time translation in action
- Performance metrics and analytics

**Recording Steps:**
1. Launch the application
2. Show the watsonx integration panel
3. Demonstrate real-time translation with live gameplay
4. Display the workflow automation
5. Show performance metrics

**Key Features to Highlight:**
- Real-time processing (<1 second latency)
- Multiple language support
- IBM watsonx workflow automation
- Quality assurance and learning
- Analytics dashboard

### Scene 4: Impact and Benefits (2:30 - 3:00)
**Visual Setup:**
- Before/after comparison
- Analytics dashboard with impressive metrics
- Team coordination improvement
- Call to action with contact information

## Technical Requirements

### Video Specifications
- **Resolution:** 1920x1080 (Full HD)
- **Frame Rate:** 30 FPS
- **Format:** MP4 (H.264)
- **Audio:** 48kHz, 16-bit, stereo
- **Duration:** Maximum 3 minutes (strict limit)

### Audio Requirements
- Clear narration with no background noise
- Balanced audio levels (narration vs. game audio)
- Professional voice-over quality
- Subtitles for accessibility

### Visual Quality
- High-resolution screen recordings
- Smooth transitions between scenes
- Consistent branding and styling
- Readable text and UI elements

## Script Timing Guide

| Time | Scene | Key Points |
|------|-------|------------|
| 0:00-0:30 | Introduction | Problem statement, solution overview, IBM watsonx mention |
| 0:30-1:00 | Problem | Show language barriers in gaming, missed communications |
| 1:00-1:30 | Solution Setup | Launch app, show watsonx integration |
| 1:30-2:15 | Live Demo | Real-time translation demonstration |
| 2:15-2:30 | Technical Details | Show watsonx workflow, performance metrics |
| 2:30-3:00 | Impact | Benefits, statistics, call to action |

## Post-Production Tips

### Editing
- Keep transitions smooth and professional
- Add subtle background music
- Include text overlays for key points
- Ensure consistent pacing throughout

### Branding
- Include IBM watsonx logo and branding
- Use consistent color scheme
- Add professional intro/outro
- Include contact information

### Quality Check
- Verify audio levels are balanced
- Check for any technical glitches
- Ensure all text is readable
- Test on different devices

## Backup Plan
- Have pre-recorded segments ready
- Prepare screenshots for technical issues
- Keep a shorter version ready (2:30) as backup
- Test all recordings before final submission

## File Naming and Submission
- **File Format:** MP4
- **File Name:** `CSGO2_Voice_Translation_IBM_Watsonx_Demo.mp4`
- **File Size:** Keep under 500MB for easy upload
- **Platform:** Upload to YouTube (unlisted) or Vimeo
- **URL:** Ensure publicly accessible link

## Final Checklist Before Recording
- [ ] IBM watsonx workspace is ready
- [ ] Application is fully configured
- [ ] Audio equipment is tested
- [ ] Recording software is configured
- [ ] Script is rehearsed and timed
- [ ] Backup recordings are prepared
- [ ] Upload destination is ready

## Common Mistakes to Avoid
- Going over 3-minute time limit
- Poor audio quality
- Unclear demonstration of IBM watsonx integration
- Technical glitches during recording
- Too much text on screen
- Inconsistent branding

Remember: Judges will not watch more than 3 minutes, so make every second count!
